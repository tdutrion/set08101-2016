There are no web standards

Only recommendations

W3C: World Wide Web Consortium

  • Founded in 1994 by Tim Berners-Lee, inventor of the World Wide Web in 1989.
  • Support a vision (Rich Interaction, offering data and services, safe place) by formalising design principles (web for all / on everything).
  • Offers technical recommendations that are / should be used as standards.

HTML and xHTML

Decribes how web pages structure should be formatted. This is used by web browser vendors to create their rendering software.

Focuses on both technical (syntax) and semantic aspects.

xHTML is the strictly formatted version of HTML, including a Document Type Definition (DTD) against which the code can be checked easily.

A list of doctype declaration is available on W3Schools if you want to have a look at different DTD.

Accessibility

In U.K., the main law on web accessibility is the Equality Act 2010

Failure to comply may be legally considered as your responsibility as professional web development practitioner.

According to the government service design manual on accessibility, you should aim for a WAI WCAG 2.0 AA validation.

For other countries, please refer to Powermapper government accessibility standards list.
